# 회원 태그 Needs Review

## Needs Review

### 정책 결정 필요

1. 태그명 길이 제한
   - 분류: 정책 결정 필요
   - 확인된 코드 동작: 태그 입력, 새 태그 추가, 태그명 수정 로직에서 앞뒤 공백 제거와 연속 공백 정리는 수행하지만 최대 글자 수 제한은 정의하지 않는다.
   - 확인할 제품 정책/의도: 회원 태그명을 몇 자까지 허용할지 결정이 필요하다.
   - 추천 기본값: LoFi 프로토타입 기준으로 현재처럼 길이 제한 없이 동작하게 두되, 운영 정책 수립 시 20자 제한을 검토한다.
   - 상태: 확인 필요

2. 태그 선택 개수 제한
   - 분류: 정책 결정 필요
   - 확인된 코드 동작: 보호자 태그, 반려견 태그, 회원/예약 필터의 선택 태그 개수 제한은 정의하지 않는다.
   - 확인할 제품 정책/의도: 한 회원 또는 필터에서 태그를 최대 몇 개까지 선택할 수 있는지 결정이 필요하다.
   - 추천 기본값: 현재 프로토타입처럼 선택 개수 제한 없이 허용한다.
   - 상태: 확인 필요

3. 저장 실패 처리 범위
   - 분류: 정책 결정 필요
   - 확인된 코드 동작: 태그 관리 저장 실패는 `태그 저장에 실패했습니다. 다시 시도해 주세요.` 토스트를 표시한다. 등록/수정 저장 흐름의 실패 처리 UI는 별도 정책으로 정의되어 있지 않다.
   - 확인할 제품 정책/의도: 등록/수정 중 태그 저장 또는 회원 저장 실패 시 현재 화면 유지, 재시도, 오류 메시지 표시 방식을 확정해야 한다.
   - 추천 기본값: 현재 화면을 유지하고 오류 토스트를 표시한다.
   - 상태: 확인 필요

### 정책 의도 확인

1. 다중 태그 필터 조건
   - 분류: 정책 의도 확인
   - 확인된 코드 동작: 회원 목록, 유치원 예약, 호텔링 예약 필터는 선택한 모든 태그가 보호자 태그와 반려견 태그를 합친 목록에 포함될 때만 결과를 표시한다.
   - 확인할 제품 정책/의도: 다중 태그 필터를 AND 조건으로 확정할지 확인이 필요하다.
   - 추천 기본값: AND 조건으로 유지한다.
   - 상태: 확인 필요

2. 태그 관리 삭제 전파
   - 분류: 정책 의도 확인
   - 확인된 코드 동작: 태그 관리에서 삭제 상태로 저장한 태그는 공통 태그 목록과 모든 회원의 보호자/반려견 태그에서 제거된다.
   - 확인할 제품 정책/의도: 태그 삭제가 전체 회원 데이터에 즉시 전파되는 동작을 제품 정책으로 확정할지 확인이 필요하다.
   - 추천 기본값: 전체 회원 데이터에서 제거한다.
   - 상태: 확인 필요

3. 태그명 수정 전파
   - 분류: 정책 의도 확인
   - 확인된 코드 동작: 태그 관리에서 태그명을 수정해 저장하면 모든 회원의 보호자/반려견 태그가 수정된 이름으로 갱신된다.
   - 확인할 제품 정책/의도: 태그명 수정이 전체 회원 데이터에 즉시 전파되는 동작을 제품 정책으로 확정할지 확인이 필요하다.
   - 추천 기본값: 전체 회원 데이터에 수정 결과를 반영한다.
   - 상태: 확인 필요
