# 회원 태그 기능 명세 (v2)

## Spec Panel 1

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 등록 |
| 화면 경로 | 회원 > 회원 등록 |
| Case | 반려견 정보 영역에서 태그를 입력하거나 기존 태그를 선택한다 |

### Spec Description 1: 태그 입력 영역

| Item | Content |
|---|---|
| Description Type | Section |
| Area | 반려견 정보 태그 필드 |

- 반려견 정보 입력 영역에 `태그` 라벨과 태그 입력 컨트롤을 표시한다.
- 선택된 태그가 있으면 입력 영역 앞에 태그 칩 목록을 표시한다.
- 선택된 태그가 없으면 선택 영역에 `태그` placeholder를 표시한다.

### Spec Description 2: 태그 입력과 추천

| Item | Content |
|---|---|
| Description Type | Input |
| Area | 태그 입력 필드 |

- 입력 필드의 placeholder는 `태그 입력`이다.
- 입력 필드에 포커스하면 추천 태그 목록을 표시한다.
- 입력값이 있으면 저장된 태그 중 선택되지 않은 태그를 실시간으로 필터링한다.
- 추천 태그는 시작 일치, 한글 초성/부분 조합 일치, 포함 일치 순서로 우선 노출한다.
- 입력값과 동일한 태그가 없으면 `"입력값" 추가` 항목을 표시한다.

### Spec Description 3: 태그 선택과 해제

| Item | Content |
|---|---|
| Description Type | Option |
| Area | 추천 태그 목록과 선택 태그 칩 |

- 추천 태그를 선택하면 해당 태그가 선택 태그 칩 목록에 추가된다.
- 선택 태그 칩의 삭제 버튼을 누르면 해당 태그가 선택 목록에서 제거된다.
- 선택 가능한 태그가 없으면 `추천 태그가 없습니다`를 표시한다.

## Spec Panel 2

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | App |
| 유저 | 비즈 |
| 화면명 | 태그 검색 |
| 화면 경로 | 회원 > 회원 등록 > 태그 |
| Case | 모바일 전체 화면에서 태그를 조회하고 다중 선택한다 |

### Spec Description 1: 태그 검색 화면 진입

| Item | Content |
|---|---|
| Description Type | Navigation |
| Area | 태그 입력 컨트롤 |

- 모바일에서 태그 입력 컨트롤을 누르면 `태그` 타이틀의 전체 화면 검색 화면으로 이동한다.
- 상단 뒤로가기 버튼을 누르면 현재 선택된 태그 상태를 유지하고 이전 화면으로 돌아간다.

### Spec Description 2: 모바일 태그 조회

| Item | Content |
|---|---|
| Description Type | Input |
| Area | 태그 검색 입력 필드 |

- 검색 입력 필드의 placeholder는 `태그 입력 또는 조회`이다.
- 검색어를 입력하면 저장된 태그 목록을 실시간으로 필터링한다.
- 검색어와 동일한 저장 태그가 없으면 `"검색어" 추가` 항목을 목록에 표시한다.

### Spec Description 3: 모바일 다중 선택 목록

| Item | Content |
|---|---|
| Description Type | Option |
| Area | 태그 체크 옵션 목록 |

- 저장된 태그는 체크 옵션 목록으로 표시한다.
- 옵션을 선택하면 체크 상태가 되고, 다시 선택하면 선택 해제된다.
- 선택된 태그는 검색 입력 영역 위에 칩으로 표시한다.
- 선택 태그 칩의 삭제 버튼을 누르면 해당 태그가 선택 해제된다.
- 선택된 태그 전체 삭제 버튼을 누르면 모든 선택 태그가 해제된다.
- 저장된 태그가 없으면 `등록된 태그가 없습니다`를 표시한다.
- 검색 결과가 없으면 `조건과 일치하는 결과가 없습니다`를 표시한다.

## Spec Panel 3

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 목록 |
| 화면 경로 | 회원 > 회원 목록 |
| Case | 회원 목록에서 태그를 다중 선택해 회원을 필터링한다 |

### Spec Description 1: 태그 필터 버튼

| Item | Content |
|---|---|
| Description Type | Section |
| Area | 목록 상단 필터 영역 |

- 목록 상단 필터 영역에 `태그` 필터 버튼을 표시한다.
- 선택된 태그가 없으면 버튼 텍스트는 `태그`이다.
- 태그를 1개 선택하면 선택한 태그명을 버튼에 표시한다.
- 태그를 2개 이상 선택하면 첫 번째 태그명과 추가 선택 개수를 함께 표시한다.

### Spec Description 2: 태그 다중 선택 메뉴

| Item | Content |
|---|---|
| Description Type | Option |
| Area | 태그 필터 드롭다운 |

- 필터 버튼을 누르면 태그 다중 선택 드롭다운을 표시한다.
- 드롭다운 상단 검색 필드의 placeholder는 `태그 입력 또는 조회`이다.
- 검색어를 입력하면 저장된 태그 목록을 실시간으로 필터링한다.
- 태그 옵션을 선택하거나 해제할 때마다 회원 목록 결과가 즉시 갱신된다.
- `초기화` 버튼을 누르면 선택된 모든 태그 조건이 해제된다.
- 저장된 태그가 없으면 `등록된 태그가 없습니다`를 표시한다.
- 검색 결과가 없으면 `조건과 일치하는 결과가 없습니다`를 표시한다.

### Spec Description 3: 회원 목록 필터 결과

| Item | Content |
|---|---|
| Description Type | List |
| Area | 회원 목록 |

- 선택한 모든 태그를 포함한 회원만 목록에 표시한다.
- 회원의 필터 대상 태그는 반려견 태그 목록을 기준으로 한다.
- 조건과 일치하는 회원이 없으면 `조건과 일치하는 결과가 없습니다.`를 표시한다.
- 회원 행에는 반려견 태그 칩을 최대 3개까지 표시하고, 초과분은 `+N` 칩으로 표시한다.

## Spec Panel 5

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 태그 관리 |
| 화면 경로 | 설정 > 태그 관리 |
| Case | 저장된 태그를 조회, 추가, 수정, 삭제하고 일괄 저장한다 |

### Spec Description 1: 태그 관리 화면

| Item | Content |
|---|---|
| Description Type | Section |
| Area | 태그 관리 본문 |

- 화면 제목은 `태그 관리`이다.
- 설명 문구는 `회원 태그를 등록하고 관리합니다.`이다.
- 저장된 전체 태그를 정렬된 목록으로 표시한다.
- 저장된 태그가 없으면 `등록된 태그가 없습니다`를 표시한다.

### Spec Description 2: 태그 검색과 추가

| Item | Content |
|---|---|
| Description Type | Input |
| Area | 태그 관리 검색 필드 |

- 검색 필드의 placeholder는 `태그 입력 또는 조회`이다.
- 검색어를 입력하면 저장된 태그 중 검색어를 포함하는 태그만 표시한다.
- 검색어와 동일한 태그가 없으면 `"검색어" 추가` 항목을 표시한다.
- `"검색어" 추가` 항목을 누르면 태그 목록에 임시 추가하고 검색어를 초기화한다.
- 검색 결과가 없으면 `조건과 일치하는 결과가 없습니다`를 표시한다.

### Spec Description 3: 태그명 수정과 삭제 상태

| Item | Content |
|---|---|
| Description Type | State |
| Area | 개별 태그 행 |

- 각 태그 행은 `태그명` placeholder를 가진 입력 필드로 태그명을 수정할 수 있다.
- 태그명을 변경한 뒤 포커스를 해제하거나 Enter를 입력하면 수정사항을 임시 반영한다.
- 삭제 버튼을 누르면 해당 행은 삭제 예정 상태가 되고 입력 필드는 비활성화된다.
- 삭제 예정 상태에서는 삭제 버튼이 `복구` 버튼으로 전환된다.
- `복구` 버튼을 누르면 삭제 예정 상태가 해제되고 입력 필드가 다시 활성화된다.

### Spec Description 4: 변경사항 저장과 이탈 경고

| Item | Content |
|---|---|
| Description Type | Button |
| Area | 하단 저장 영역과 이탈 경고 |

- 추가, 수정, 삭제 변경사항이 있으면 하단 저장 영역을 표시한다.
- `저장` 버튼을 누르면 변경사항을 일괄 저장한다.
- 저장에 성공하면 `저장했습니다.` 토스트를 표시한다.
- 저장하지 않은 변경사항이 있는 상태에서 다른 메뉴로 이동하면 `변경을 취소하시겠습니까?\n변경한 설정은 저장되지 않습니다.` 경고를 표시한다.
- 경고에서 `변경 취소`를 선택하면 변경사항을 저장하지 않고 이동한다.
- 경고에서 `닫기`를 선택하면 태그 관리 화면에 머문다.

## Policy Panel

**Policy: 태그 입력 및 카탈로그 제한**

- affected scope: 회원 등록, 회원 수정, 태그 관리, 예약 필터
- confirmed policy: 한 회원 또는 한 반려견에 동시에 지정할 수 있는 태그는 최대 10개이다.
- confirmed policy: 전체 태그 카탈로그에는 최대 20개까지 태그를 등록할 수 있다.
- confirmed policy: 태그명은 앞뒤 공백을 제거하고 연속 공백을 한 칸으로 정규화한다.
- confirmed policy: 태그 중복 비교는 대소문자와 중복 공백 차이를 무시한다.
- confirmed policy: 저장되는 태그 목록은 한국어 정렬 기준으로 정렬한다.

**Policy: 태그 검색 및 필터 정책**

- affected scope: 태그 입력 추천, 태그 관리 검색, 회원 목록 필터, 예약 목록 필터
- confirmed policy: 추천 태그는 선택되지 않은 저장 태그만 대상으로 한다.
- confirmed policy: 태그 추천은 시작 일치, 한글 초성/부분 조합 일치, 포함 일치 순서로 정렬한다.
- confirmed policy: 다중 태그 필터는 선택한 모든 태그를 포함해야 통과하는 AND 조건이다.
- confirmed policy: 유치원/호텔 예약 필터는 예약에 연결된 반려견 태그 목록을 기준으로 한다.

**Policy: 태그 수정 및 삭제 전파**

- affected scope: 태그 관리, 회원 상세, 회원 등록/수정, 예약 목록
- confirmed policy: 태그명을 수정하면 기존 회원과 반려견에 연결된 동일 태그명이 함께 갱신된다.
- confirmed policy: 태그를 삭제하면 기존 회원과 반려견에 연결된 동일 태그명이 함께 제거된다.
- confirmed policy: Web 태그 관리는 변경사항을 임시 보관한 뒤 `저장` 버튼으로 일괄 반영한다.
- confirmed policy: App 태그 관리는 추가, 수정, 삭제 동작을 즉시 저장한다.
