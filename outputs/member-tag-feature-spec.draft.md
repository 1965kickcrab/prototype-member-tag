# 회원 태그 기능 명세 Draft

## Spec Panel 1

**META / Case**

| 항목 | 내용 |

| 유저 | 비즈 |
| 화면명 | 회원 등록 |
| 화면 경로 | 회원 > 회원 등록 |
| Case | 반려견 정보 또는 보호자 수정 영역에서 태그를 입력하거나 추천 태그를 선택한다 |

**Policy 태그 정규화**

- 태그 입력값은 앞뒤 공백을 제거하고 연속 공백을 한 칸으로 정리한다.
- 빈 값은 태그로 추가하지 않는다.
- 이미 선택된 태그와 대소문자만 다른 값은 중복 추가하지 않는다.
- 태그 저장 시 중복 태그는 대소문자를 무시하고 하나만 유지한다.
- 태그명 길이 제한과 선택 개수 제한은 확인 필요이다.

**Section 태그 입력 영역**

- 반려견 정보 입력 영역에는 `태그` 항목이 표시된다.
- 보호자 수정 영역에는 주소 입력 다음에 `태그` 항목이 표시된다.
- 선택한 태그가 있으면 입력 필드 앞에 태그 칩으로 표시된다.
- 선택한 태그가 없으면 태그 칩 영역은 빈 상태로 유지된다.

**Input 태그 입력**

- 입력 필드 placeholder는 `태그 입력`이다.
- 입력 필드에 focus하거나 입력값을 변경하면 추천 태그 메뉴가 열린다.
- Enter를 누르면 첫 번째 추천 태그가 선택된다.
- 추천 태그가 없으면 현재 입력값이 새 태그로 추가된다.
- 한글 조합 중 Enter 입력은 태그 추가로 처리하지 않는다.

**Option 추천 태그**

- 추천 태그는 저장된 태그 목록 중 현재 선택되지 않은 태그만 표시한다.
- 검색어가 없으면 저장된 태그 목록 순서대로 최대 6개를 표시한다.
- 검색어가 있으면 시작 일치, 한글 초성/부분 입력 일치, 포함 일치 순서로 추천한다.
- 검색어와 같은 저장 태그가 추천 목록에 없으면 `"입력값" 추가` 옵션을 표시한다.
- 추천 항목이 없고 입력값도 없으면 `추천 태그가 없습니다`를 표시한다.

**Button 태그 선택 및 해제**

- 추천 태그 또는 `"입력값" 추가` 옵션을 선택하면 해당 태그가 선택 태그에 추가되고 입력값은 비워진다.
- 태그 칩의 `×` 버튼을 누르면 해당 태그만 선택 목록에서 제거된다.
- 선택 태그가 있을 때 표시되는 `×` 전체 해제 버튼을 누르면 선택한 태그가 모두 제거된다.

**META / Case**

| 화면 경로 | 회원 > 회원 등록 > 태그 |
| Case | 모바일 태그 검색 화면에서 저장된 태그를 조회하거나 새 태그를 선택한다 |

**Navigation 태그 검색 화면**

- 모바일에서 태그 입력 필드에 focus하면 전체 화면 태그 검색 화면이 열린다.
- 태그 검색 화면 상단에는 뒤로가기 버튼과 제목 `태그`가 표시된다.
- 뒤로가기 버튼을 누르면 태그 검색 화면을 닫고 기존 입력 영역으로 돌아간다.

**Input 태그 입력 또는 조회**

- 검색 입력 placeholder는 `태그 입력 또는 조회`이다.
- 검색어가 없으면 저장된 태그 목록 전체를 표시한다.
- 검색어가 있으면 저장된 태그 중 검색어를 포함하는 태그만 표시한다.
- 한글 조합 중에는 목록 영역만 갱신하고 조합 완료 후 입력 focus를 유지한다.

**Option 태그 목록**

- 각 태그는 체크박스 옵션으로 표시된다.
- 체크하면 선택 태그 칩에 추가되고, 체크 해제하면 선택 목록에서 제거된다.
- 검색어와 같은 저장 태그가 없으면 `"입력값" 추가` 체크 옵션을 표시한다.
- 검색어가 있고 결과가 없으면 `조건과 일치하는 결과가 없습니다`를 표시한다.
- 저장된 태그가 하나도 없고 검색어가 없으면 `등록된 태그가 없습니다`를 표시한다.

**Button 선택 태그 해제**

- 선택한 태그는 검색 입력 앞에 칩으로 표시된다.
- 태그 칩의 `×` 버튼을 누르면 해당 태그만 제거된다.
- 선택 태그가 있을 때 전체 해제 `×` 버튼을 누르면 선택한 태그가 모두 제거되고 검색 화면은 열린 상태를 유지한다.

## Spec Panel 3

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 등록 |
| 화면 경로 | 회원 > 회원 등록 |
| Case | 회원 등록을 완료해 보호자 태그와 반려견 태그를 저장한다 |

**Button 등록**

- `등록` 버튼은 등록 가능한 반려견 정보가 1개 이상 있을 때 활성화된다.
- `등록` 버튼을 누르면 `회원 등록 후 회원에게 등록 안내 알림톡을 보내시겠습니까?` 알림이 표시된다.
- 알림의 확인 버튼 `등록`을 누르면 보호자 태그와 반려견 태그가 회원 데이터에 저장된다.
- 저장 후 회원 목록으로 이동하고 `회원을 등록했습니다.` 토스트를 표시한다.
- 등록 저장 실패 처리 정책은 확인 필요이다.

**Policy 태그 저장 데이터**

- 보호자 태그는 보호자 정보에 저장된다.
- 반려견 태그는 각 반려견 정보에 분리 저장된다.
- 저장된 보호자 태그와 모든 반려견 태그는 공통 태그 목록에 병합된다.
- 공통 태그 목록은 중복 제거된 목록으로 유지된다.
- 같은 전화번호 또는 같은 회원 ID의 기존 회원이 있으면 기존 회원 데이터와 병합된다.

**Value 태그 반영**

- 등록 후 보호자 태그는 보호자 정보 표시와 예약/회원 필터에 사용할 수 있다.
- 등록 후 반려견 태그는 회원 목록의 반려견 행과 예약/회원 필터에 사용할 수 있다.

## Spec Panel 4

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 목록 |
| 화면 경로 | 회원 > 회원 목록 |
| Case | 회원 목록에서 태그를 여러 개 선택해 회원을 필터링한다 |

**Section 태그 필터**

- 회원 목록 필터 영역에는 태그 필터 버튼과 `초기화` 버튼이 표시된다.
- 태그가 선택되지 않았을 때 필터 버튼 텍스트는 `태그`이다.
- 태그 1개를 선택하면 필터 버튼에 선택한 태그명이 표시된다.
- 태그를 2개 이상 선택하면 `첫 번째 태그 +추가 개수` 형식으로 표시된다.

**Option 태그 다중 선택**

- 태그 필터 버튼을 누르면 `태그 검색` 입력과 태그 체크박스 목록이 포함된 메뉴가 열린다.
- `태그 검색` 입력값을 포함하는 태그가 대소문자 구분 없이 표시된다.
- 저장된 태그가 없으면 `등록된 태그가 없습니다`를 표시한다.
- 검색 결과가 없으면 `조건과 일치하는 결과가 없습니다`를 표시한다.
- 여러 태그를 동시에 선택할 수 있다.

**List 필터 결과**

- 선택한 태그가 없으면 태그 조건 없이 회원 목록을 표시한다.
- 태그가 선택되면 보호자 태그 또는 반려견 태그를 합친 목록에 선택한 태그를 모두 포함하는 회원 행만 표시한다.
- 다중 태그 필터 조건은 제품 의도 확인 필요이다.
- 반려견 행에는 반려견 태그를 최대 3개까지 칩으로 표시한다.
- 표시 가능한 태그보다 더 많은 태그가 있으면 `+남은 개수` 칩을 표시한다.
- 필터 결과가 없으면 `조건과 일치하는 결과가 없습니다.`를 표시한다.

**Button 초기화**

- `초기화` 버튼을 누르면 선택 태그와 태그 검색어가 모두 비워진다.
- 초기화 후 태그 메뉴는 닫히고 회원 목록 첫 페이지로 돌아간다.

**Section 태그 바텀시트**

- 모바일 회원 목록 상단에는 태그 필터 버튼과 회원 검색 입력이 함께 표시된다.
- 태그 필터 버튼을 누르면 제목 `태그`가 있는 바텀시트가 열린다.
- 바텀시트 헤더에는 태그 관리 아이콘 버튼과 `닫기` 버튼이 표시된다.
- `닫기` 버튼을 누르면 바텀시트가 닫힌다.

**Input 태그 검색**

- 바텀시트 검색 입력 placeholder는 `태그 검색`이다.
- 선택 태그가 있으면 검색 입력 앞에 선택 태그 칩이 표시된다.
- 선택 태그가 있으면 전체 해제 `×` 버튼이 표시되고, 누르면 선택 태그와 검색어가 모두 비워진다.

**List 필터 결과**

- 태그 선택 방식과 회원 목록 필터 결과 규칙은 Web 회원 목록 태그 필터와 동일하다.
- 바텀시트에서 태그를 선택하거나 해제하면 회원 목록이 즉시 갱신된다.

## Spec Panel 6

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 태그 관리 |
| 화면 경로 | 회원 > 회원 목록 > 태그 관리 |
| Case | 저장된 태그명을 수정하거나 삭제 상태로 바꾸고 전체 회원 데이터에 반영한다 |

**Section 태그 관리 모달**

- 태그 관리 진입 시 `태그 관리` 모달이 열린다.
- 모달에는 `태그 검색` 입력, 태그 편집 목록, 하단 `초기화` 버튼과 `저장` 버튼이 표시된다.
- 닫기 버튼을 누르면 편집 중인 변경사항을 버리고 모달을 닫는다.

**Input 태그 검색 및 태그명**

- 검색 입력 placeholder는 `태그 검색`이다.
- 검색어는 기존 태그명 또는 수정 예정 태그명에 포함되는 태그 행을 표시한다.
- 각 태그 행의 입력 placeholder는 `태그명`이다.
- 태그명 입력값을 변경하면 해당 태그의 저장 예정 이름이 변경된다.
- 삭제 상태인 태그 행의 입력 필드는 비활성화된다.

**Button 추가, 삭제, 복원**

- 검색어와 같은 태그가 편집 목록에 없으면 `"검색어" 추가` 행이 표시된다.
- `"검색어" 추가` 행을 누르면 해당 태그가 편집 목록에 추가되고 검색어는 비워진다.
- 일반 상태 태그 행에는 `삭제` 버튼이 표시된다.
- `삭제` 버튼을 누르면 해당 행이 삭제 상태가 되고 버튼명이 `복원`으로 바뀐다.
- `복원` 버튼을 누르면 해당 행이 다시 편집 상태가 된다.

**Button 저장**

- `저장` 버튼을 누르면 태그명 변경과 삭제 상태가 공통 태그 목록에 반영된다.
- 태그명 변경은 모든 회원의 보호자 태그와 반려견 태그에도 동일하게 반영된다.
- 삭제 상태 태그는 공통 태그 목록과 모든 회원의 보호자/반려견 태그에서 제거된다.
- 태그명 수정과 삭제의 전체 전파 정책은 제품 의도 확인 필요이다.
- 현재 선택 중인 회원 목록 필터 태그도 수정/삭제 결과에 맞게 갱신된다.
- 저장 성공 시 모달이 닫히고 `태그를 저장했습니다.` 토스트를 표시한다.
- 저장 실패 시 `태그 저장에 실패했습니다. 다시 시도해 주세요.` 토스트를 표시한다.

**State 빈 목록**

- 저장된 태그가 없고 추가 가능한 검색어도 없으면 `등록된 태그가 없습니다`를 표시한다.
- 검색 결과가 없고 추가 가능한 검색어도 없으면 `조건과 일치하는 결과가 없습니다`를 표시한다.

## Spec Panel 7

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 상세 |
| 화면 경로 | 회원 > 회원 상세 |
| Case | 회원 상세에서 보호자 태그와 반려견 태그를 확인하고 수정한다 |

**Section 태그 표시**

- 회원 상세 상단 프로필에는 반려견 태그가 있을 때 태그 칩을 표시한다.
- `보호자 정보` 카드에는 보호자 태그가 있을 때 카드 헤더에 태그 칩을 표시한다.
- 태그가 없으면 해당 태그 칩 영역은 표시하지 않는다.

**Button 보호자 수정**

- `보호자 정보`의 `수정`을 누르면 보호자 수정 모달이 열린다.
- 보호자 수정 모달의 `태그` 항목에서 보호자 태그를 수정할 수 있다.

- 예약 검색 필터에는 예약자/반려견 검색 입력과 태그 필터 버튼이 함께 표시된다.
- 태그 필터 버튼의 기본 텍스트는 `태그`이다.
- 태그 1개를 선택하면 선택한 태그명이 표시된다.
- 태그를 2개 이상 선택하면 `첫 번째 태그 +추가 개수` 형식으로 표시된다.

**Option 태그 다중 선택**

- 태그 필터 버튼을 누르면 `태그 검색` 입력과 태그 체크박스 목록이 열린다.
- 저장된 태그가 없으면 `등록된 태그가 없습니다`를 표시한다.
- 검색 결과가 없으면 `조건과 일치하는 결과가 없습니다`를 표시한다.
- 선택 태그가 있으면 선택 태그 칩과 전체 해제 `×` 버튼이 표시된다.

**List 예약 필터 결과**

- 예약 데이터는 저장된 회원 데이터와 연결되어 보호자 태그와 반려견 태그를 함께 가진다.
- 태그가 선택되면 예약의 보호자 태그와 반려견 태그를 합친 목록에 선택한 태그가 모두 포함된 예약만 표시한다.
- 태그 조건은 현재 선택 날짜의 예약 목록과 예약 검색어 조건에 함께 적용된다.
