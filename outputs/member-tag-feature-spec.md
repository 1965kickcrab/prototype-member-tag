# 회원 태그 기능 명세

## Spec Panel 1

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 등록 |
| 화면 경로 | 회원 > 회원 등록 |
| Case | 반려견 정보 또는 보호자 정보 영역에서 태그를 입력하거나 추천 태그를 선택한다 |

### Spec Description 1: 태그 입력 영역 표시

| Item | Content |
|---|---|
| Description Type | Section |
| Area | 반려견 정보 및 보호자 정보의 태그 영역 |

- 반려견 정보 입력 영역 및 보호자 정보 수정 영역에 "태그" 입력 영역이 노출된다.
- 선택된 태그가 존재할 경우, 태그 입력창 왼쪽에 태그 칩 리스트 형태로 표시된다.
- 선택된 태그가 없는 경우, 빈 상태로 표시된다.

### Spec Description 2: 태그 입력 및 제안

| Item | Content |
|---|---|
| Description Type | Input |
| Area | 태그 입력 필드 |

- 입력창의 기본 placeholder 텍스트는 "태그 입력"이다.
- 입력 필드에 마우스 포커스 시 추천 태그 레이어가 하단에 즉시 열린다.
- 입력창에 키워드를 입력하면 입력값에 해당하는 추천 태그가 실시간 필터링되어 나타난다.
- 태그 입력값은 최대 10자까지 허용하며, 10자 초과 입력 시 더 이상 입력되지 않는다.
- 한글 자모가 아직 조합 중인 상태에서 Enter 입력은 태그 등록으로 처리하지 않는다.

### Spec Description 3: 추천 태그 선택

| Item | Content |
|---|---|
| Description Type | Option |
| Area | 추천 태그 레이어 |

- 추천 태그 레이어에는 전체 태그 중 현재 선택되지 않은 태그들이 최대 6개까지 노출된다.
- 검색어가 없는 상태에서는 저장된 전체 태그 목록이 정렬 순으로 노출된다.
- 검색어가 입력되면 시작 일치, 초성 일치, 부분 포함 순서로 추천 가중치를 두어 정렬 및 노출한다.
- 검색어와 완벽히 일치하는 태그가 없을 경우, 추천 목록 맨 위에 `"{검색어} 추가"` 항목을 노출한다.
- 전체 등록된 태그가 없고 검색어도 입력되지 않은 상태에서는 "등록된 태그가 없습니다"가 표시된다.

### Spec Description 4: 태그 추가 및 해제

| Item | Content |
|---|---|
| Description Type | Button |
| Area | 태그 칩 및 제안 항목 |

- 추천 태그 항목 또는 `"{검색어} 추가"` 항목을 클릭하거나 Enter를 입력하면 해당 태그가 선택 칩 리스트에 추가되고 입력 필드는 초기화된다.
- 태그 칩 우측의 `×` 버튼을 누르면 해당 태그가 선택 리스트에서 제거된다.
- 한 회원에게 부여된 태그가 10개 이상인 상태에서 태그 추가 시도 시 "한 회원당 최대 10개까지만 태그를 붙일 수 있습니다." 토스트를 표시하고 추가하지 않는다.
- 추가하려는 태그가 신규 태그이고 전체 태그 카탈로그 크기가 20개 이상일 경우, "태그는 최대 20개까지 등록할 수 있습니다." 토스트를 표시하고 추가하지 않는다.

### Spec Description 1: 모바일 태그 검색 화면 전환

| Item | Content |
|---|---|
| Description Type | Navigation |
| Area | 태그 입력 포커스 시 검색 전체 화면 |

- 모바일 기기에서 회원 정보의 태그 입력란을 터치하면 헤더에 뒤로가기 버튼과 "태그" 타이틀을 갖는 검색 전용 풀스크린 화면으로 진입한다.
- 좌측 상단의 뒤로가기 버튼을 누르면 선택된 태그 목록 상태를 가지고 이전 입력 화면으로 복귀한다.

### Spec Description 2: 검색 필드 입력

| Item | Content |
|---|---|
| Description Type | Input |
| Area | 모바일 태그 검색창 |

- 검색창의 placeholder 텍스트는 "태그 입력 또는 조회"이다.
- 태그 입력값은 최대 10자까지 허용하며, 10자 초과 입력 시 더 이상 입력되지 않는다.
- 한글 완성 이전(자모 조합 중) 상태에서도 목록 검색 결과가 즉시 연동되며 입력 포커스는 해제되지 않는다.

### Spec Description 3: 체크박스 옵션 목록

| Item | Content |
|---|---|
| Description Type | Option |
| Area | 태그 체크박스 목록 |

- 저장된 전체 태그 목록이 하단에 스크롤 가능한 형태의 체크박스 옵션 목록으로 노출된다.
- 각 항목 선택 시 체크(선택 상태) 및 해제(선택 해제) 토글이 적용된다.
- 검색어를 입력했으나 일치하는 저장 태그가 존재하지 않을 경우, 목록 최상단에 `"{검색어} 추가"` 체크 옵션을 노출한다.
- 검색어에 필터링된 결과가 없을 경우 "조건과 일치하는 결과가 없습니다" 문구를 표시한다.
- 전체 등록된 태그가 없고 검색어도 없는 초기 상태인 경우 "등록된 태그가 없습니다" 문구를 노출한다.

### Spec Description 4: 선택 태그 관리

| Item | Content |
|---|---|
| Description Type | Button |
| Area | 상단 선택 태그 칩 리스트 |

- 선택된 태그들은 입력창 위에 칩 목록 형태로 노출된다.
- 개별 태그 칩의 `×` 버튼을 터치하면 선택이 취소된다.
- 선택 리스트 우측 끝의 전체 해제 `×` 버튼 터치 시 모든 태그 선택이 일괄 해제된다.
- 선택 중인 태그 개수가 10개 이상일 경우 신규 추가 시도 시 "한 회원당 최대 10개까지만 태그를 붙일 수 있습니다." 토스트를 표시한다.
- 신규 생성하려는 태그가 전체 등록 한도(20개)를 초과할 경우 "태그는 최대 20개까지 등록할 수 있습니다." 토스트를 표시한다.

### Spec Description 1: 등록 저장 실행

| Item | Content |
|---|---|
| Description Type | Button |
| Area | 회원 등록 화면 하단 등록 버튼 |

- 반려견 정보 및 보호자 정보가 최소한의 검증 조건(이름, 연락처 등)을 충족하여 작성되면 하단의 "등록" 버튼이 활성화된다.
- "등록" 버튼을 클릭하면 확인 팝업("회원 등록 후 회원에게 등록 안내 알림톡을 보내시겠습니까?")이 출력된다.
- 확인 팝업 내 "등록"을 클릭하면 태그를 포함한 회원 전체 정보가 브라우저 로컬 저장소에 저장되고 완료 토스트("회원을 등록했습니다.")와 함께 회원 목록 화면으로 이동한다.

## Spec Panel 4

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 목록 필터 |
| 화면 경로 | 회원 > 회원 목록 |
| Case | 회원 목록에서 태그를 여러 개 선택해 회원을 필터링한다 |

### Spec Description 1: 태그 필터 영역

| Item | Content |
|---|---|
| Description Type | Section |
| Area | 목록 상단 필터 영역 |

- 목록 상단에 태그 필터용 트리거 버튼이 제공된다.
- 선택된 태그가 없을 때의 버튼 텍스트는 "태그"이다.
- 태그를 1개만 선택한 경우 해당 태그명이 버튼에 단독 표시된다.
- 태그를 2개 이상 선택한 경우 `"{첫 번째 태그명} +{추가 선택 수}"` 형식으로 노출된다.

### Spec Description 2: 태그 다중 선택 메뉴

| Item | Content |
|---|---|
| Description Type | Option |
| Area | 태그 필터 드롭다운 메뉴 |

- 트리거 버튼을 클릭하면 드롭다운 형태로 태그 다중 선택 메뉴가 열린다.
- 상단의 "태그 검색" 입력 필드를 통해 특정 태그명 검색이 가능하다.
- 검색어가 포함된 태그들이 아래 체크박스 목록으로 노출되며, 다중 선택이 가능하다.
- 저장된 태그가 없을 경우 "등록된 태그가 없습니다"가 표시된다.
- 검색어와 일치하는 결과가 없는 경우 "조건과 일치하는 결과가 없습니다"가 표시된다.

### Spec Description 3: 필터 초기화

| Item | Content |
|---|---|
| Description Type | Button |
| Area | 태그 필터 드롭다운 내 초기화 버튼 |

- 드롭다운 내 "초기화" 버튼을 클릭하면 현재 선택되었던 모든 태그 조건이 해제되고 드롭다운 메뉴가 닫히며 회원 목록 필터가 초기화된다.

### Spec Description 4: 필터 결과 목록

| Item | Content |
|---|---|
| Description Type | List |
| Area | 회원 목록 |

- 태그 필터에서 선택한 모든 태그가 보호자 태그 및 반려견 태그 전체 목록 내에 전부 포함되어 있는(AND 조건) 회원 행만 정렬되어 출력된다.
- 일치하는 결과가 없는 경우 "조건과 일치하는 결과가 없습니다." 공백 상태 메시지를 출력한다.
- 각 회원 행의 반려견 이름 영역 우측에는 부여된 태그 칩이 노출되며, 최대 3개까지 표시되고 초과할 경우 `"+{초과수}"` 칩으로 생략 노출한다.

- 선택된 태그는 칩 형태로 표시되며 개별 칩의 `×` 버튼을 터치하여 바로 삭제할 수 있다.
- 다중 선택 영역 우측 끝의 전체 해제 `×` 버튼을 터치하면 선택한 태그가 모두 비워진다.

**META / Case**

| 항목 | 내용 |

| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 태그 관리 |
| 화면 경로 | 설정 > 태그 관리 |
| Case | 저장된 태그명을 수정하거나 삭제한다 |

### Spec Description 1: 태그 관리 진입 및 종료

| Item | Content |
|---|---|
| Description Type | Navigation |
| Area | 설정 화면의 태그 관리 탭 |

- PC 웹 설정 메뉴에서 "태그 관리" 화면에 진입하면 저장된 전체 태그 목록이 카드 레이아웃으로 노출된다.
- 사이드바 메뉴를 눌러 화면을 이탈하려 할 때, 저장되지 않은 수정 내역이 있는 경우 "변경을 취소하시겠습니까?\n변경한 설정은 저장되지 않습니다."라는 경고 얼럿이 호출된다.
- 얼럿창의 "변경 취소" 선택 시 데이터 갱신을 생략하고 해당 메뉴로 이동한다. "닫기" 선택 시 원래의 태그 관리 화면에 머무른다.

### Spec Description 2: 태그명 검색 및 신규 생성

| Item | Content |
|---|---|
| Description Type | Input |
| Area | 태그 관리 상단 검색 영역 |

- 상단 검색창의 placeholder는 "태그 검색"이다.
- 입력창에 키워드 입력 시 해당 이름을 포함하는 태그 행만 리스트에 실시간 조회된다.
- 검색 키워드와 정확히 일치하는 태그가 없을 경우 최상단에 `"{검색어} 추가"` 버튼 행이 노출되며, 이를 누르면 임시 카탈로그 목록에 해당 태그가 추가되고 입력창은 초기화된다.

### Spec Description 3: 태그 수정 및 삭제 변경

| Item | Content |
|---|---|
| Description Type | State |
| Area | 개별 태그 목록 행 |

- 태그 행의 입력 필드에 기본적으로 현재 태그명이 채워져 있으며, 사용자가 이를 변경하고 포커스를 잃거나 Enter를 누르면 태그명 수정 사항이 임시 기록된다.
- 태그명 입력란에는 최대 10자까지만 입력이 가능하다.
- 우측의 "삭제" 버튼 클릭 시 해당 행은 취소선 스타일의 "삭제 상태"(`state: deleted`)로 전환되며 텍스트 입력창은 비활성화된다. "삭제" 버튼은 "복구" 버튼으로 전환된다.
- "복구" 버튼 클릭 시 원상태 복구되며 텍스트 입력창이 다시 활성화된다.
- 태그 삭제 시도 시 해당 태그가 연결된 반려견이 존재할 경우 대체 태그 지정 화면(Spec Panel 7) 모달이 먼저 출력된다.

### Spec Description 4: 일괄 저장 처리

| Item | Content |
|---|---|
| Description Type | Button |
| Area | 하단 저장 바 |

## Policy Panel

**Policy: 태그 등록 한도 및 검증 정책**

- affected scope: 회원 등록, 회원 상세 정보 수정 (Web & App)
- confirmed policy:
  - 회원 태그명은 **최대 10자** 길이로 제한한다. 10자를 초과하는 키워드는 생성 및 수정이 불가하다.
  - 회원 1인(또는 해당 소속 반려견 1마리)에게 동시 지정 가능한 최대 태그 수는 10개로 제한한다.
  - 전체 시스템에 등록되어 보관될 수 있는 고유 태그 카탈로그 한도는 최대 20개로 제한한다.
  - 태그 등록 요청 값은 항상 정규화(앞뒤 공백 트림, 중간 2칸 이상의 공백은 1칸으로 치환)를 수행하며, 대소문자 구분을 무시하고 중복 등록을 차단한다.

**Policy: 태그 삭제 및 대체 정책**

- affected scope: 설정 및 더보기 내 태그 관리 기능
- confirmed policy:
  - 시스템에 존재하는 태그 삭제를 진행할 때, 해당 태그를 사용 중인 반려견 데이터가 존재한다면 즉시 삭제 처리를 보류하고 대체 태그를 입력받는 경고 모달을 띄운다.
  - 사용자는 대체 태그 지정 필드에서 "선택 안함"을 고르거나, 기존의 다른 활성화된 태그를 검색해 매핑하거나, 즉석에서 새로운 태그명을 입력해 등록하여 대체할 수 있다.
  - 삭제를 확정 및 저장하면 대체 태그로 지정된 태그 정보가 해당 태그를 부여받아 사용 중이던 모든 회원의 정보에 즉각 전파 매핑되어 변경 처리된다.
  - 대체 태그를 "선택 안함"으로 지정하고 삭제를 승인한 경우, 해당 회원들의 반려견 정보에서 해당 태그 항목은 빈 리스트 상태로 제거 전파(일괄 제거 전파 방식)된다.
