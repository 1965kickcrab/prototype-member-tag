# 회원 태그 기능 명세

## Spec Panel 1

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 등록 |
| 화면 경로 | 회원 > 회원 등록 |
| Case | 반려견 정보에 태그를 입력하거나 추천 태그를 선택해 등록한다 |

**Section 태그 입력 영역**

• 반려견 정보 입력 영역에는 `태그` 항목이 표시된다.
• 태그 입력 필드의 placeholder는 `태그 입력`이다.
• 선택한 태그는 입력 필드 앞에 칩으로 표시된다.
• 선택한 태그가 없으면 태그 칩과 전체 해제 버튼은 표시되지 않는다.

**Input 태그 입력**

• 입력한 태그명은 앞뒤 공백이 제거되고 연속 공백은 한 칸으로 정리된다.
• 빈 값은 태그로 추가되지 않는다.
• 이미 선택한 태그와 대소문자만 다른 값은 중복 추가되지 않는다.
• Enter 키를 누르면 첫 번째 추천 태그가 추가되고, 추천 태그가 없으면 현재 입력값이 새 태그로 추가된다.

**Option 추천 태그**

• 입력 필드에 focus되거나 입력값이 변경되면 추천 태그 메뉴가 열린다.
• 추천 태그는 저장된 태그 목록 중 현재 선택되지 않은 태그에서 최대 6개까지 표시된다.
• 검색어가 없으면 저장된 태그 목록 순서대로 추천한다.
• 검색어가 있으면 태그명의 시작 일치, 한글 초성/부분 입력 일치, 포함 일치 순으로 추천한다.
• 검색어와 같은 태그가 추천 목록에 없으면 `"입력값" 추가` 옵션을 표시한다.
• 추천할 항목이 없으면 `추천 태그가 없습니다`를 표시한다.

**Button 태그 선택/삭제**

• 추천 태그를 클릭하면 해당 태그가 선택 태그로 추가되고 입력값은 비워진다.
• 태그 칩의 `×` 버튼을 누르면 해당 태그가 선택 목록에서 제거된다.
• 선택 태그가 있을 때 표시되는 `×` 전체 해제 버튼을 누르면 선택한 태그가 모두 제거된다.

## Spec Panel 2

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | App |
| 유저 | 비즈 |
| 화면명 | 회원 등록 |
| 화면 경로 | 회원 > 회원 등록 |
| Case | 모바일에서 태그 검색 화면을 열어 반려견 태그를 선택한다 |

**Navigation 태그 검색 화면**

• 모바일 폭에서 태그 입력 필드에 focus하면 별도 태그 검색 화면이 열린다.
• 태그 검색 화면 상단에는 뒤로가기 버튼과 제목 `태그`가 표시된다.
• 뒤로가기 버튼을 누르면 태그 검색 화면이 닫히고 기존 입력 영역으로 돌아간다.

**Input 태그 검색**

• 검색 입력 placeholder는 `태그 입력 또는 조회`이다.
• 검색어 입력 중에는 저장된 태그 목록이 검색어 기준으로 갱신된다.
• 검색어가 없으면 저장된 태그 목록 전체가 표시된다.
• 검색어가 있을 때 일치하는 저장 태그가 없으면 `조건과 일치하는 결과가 없습니다` 상태를 표시한다.
• 저장된 태그가 하나도 없고 검색어도 없으면 `등록된 태그가 없습니다` 상태를 표시한다.

**Option 태그 목록**

• 태그 목록 제목은 `태그 목록`이다.
• 목록의 각 태그는 체크박스로 표시된다.
• 태그를 체크하면 선택 태그 칩에 추가되고 검색 화면은 열린 상태로 유지된다.
• 선택된 태그를 다시 체크 해제하면 선택 목록에서 제거된다.
• 검색어와 같은 저장 태그가 없으면 `"입력값" 추가` 옵션을 표시하고, 선택 시 새 태그로 추가한다.

## Spec Panel 3

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 보호자 수정 |
| 화면 경로 | 회원 > 회원 등록 > 보호자 수정 |
| Case | 보호자 태그를 수정하고 회원 등록 화면에 반영한다 |

**Section 보호자 태그**

• 보호자 수정 화면에는 `태그` 항목이 표시된다.
• 보호자 성함과 전화번호는 읽기 전용이며 태그는 수정 가능하다.
• 태그 입력 방식은 회원 등록의 태그 입력 정책과 동일하다.

**Button 수정**

• `수정` 버튼을 누르면 보호자 주소, 상세 주소, 보호자 태그가 회원 등록 화면의 보호자 정보에 반영된다.
• 저장된 보호자 태그는 공통 태그 목록에 병합되어 이후 추천 태그로 사용할 수 있다.
• 보호자 태그가 없으면 회원 등록 화면의 보호자 정보 영역에 보호자 태그 섹션은 표시되지 않는다.
• 보호자 태그가 있으면 `보호자 태그` 제목과 태그 칩 목록이 표시된다.

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | App |
| 유저 | 비즈 |
| 화면명 | 반려견 추가 |
| 화면 경로 | 회원 > 회원 등록 > 반려견 추가 |
| Case | 모바일 바텀시트에서 반려견 태그를 입력하고 반려견을 추가한다 |

**Section 반려견 추가 바텀시트**

• `반려견 추가` 바텀시트에는 반려견 정보 입력 영역과 `태그` 항목이 표시된다.
• 태그 입력 방식은 모바일 태그 검색 화면 정책과 동일하다.

**Button 추가**

• `추가` 버튼은 반려견 이름과 견종이 모두 입력된 경우에만 활성화된다.
• 태그만 입력한 상태에서는 `추가` 버튼이 활성화되지 않는다.
• `추가` 버튼을 누르면 입력한 반려견 정보와 반려견 태그가 회원 등록 화면의 반려견 목록에 추가된다.
• 추가된 태그는 최종 회원 등록 전까지 현재 작성 중인 반려견 정보에만 반영된다.

## Spec Panel 5

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 등록 |
| 화면 경로 | 회원 > 회원 등록 |
| Case | 회원 등록 완료 시 보호자 태그와 반려견 태그를 저장한다 |

**Button 등록**

• `등록` 버튼은 반려견 이름과 견종이 입력된 반려견이 1개 이상 있을 때 활성화된다.
• `등록` 버튼을 누르면 `회원 등록 후 회원에게 등록 안내 알림톡을 보내시겠습니까?` 알림이 표시된다.
• 알림의 확인 버튼 `등록`을 누르면 보호자 태그와 반려견 태그가 회원 데이터에 저장된다.
• 저장 시 보호자 태그와 모든 반려견 태그는 공통 태그 목록에 병합된다.
• 저장이 끝나면 회원 목록 화면으로 이동하고 `회원을 등록했습니다.` 토스트가 표시된다.

**Policy 저장 데이터**

• 회원 데이터에는 보호자 태그와 반려견별 태그가 분리되어 저장된다.
• 공통 태그 목록은 저장된 태그를 중복 제거한 목록으로 유지된다.
• 기존 회원과 같은 보호자 전화번호 또는 같은 회원 ID로 등록하면 기존 회원 데이터와 병합된다.

## Spec Panel 6

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 목록 |
| 화면 경로 | 회원 > 회원 목록 |
| Case | 회원 목록에서 태그를 선택해 회원을 필터링한다 |

**Section 태그 필터**

• 회원 목록에는 태그 필터 버튼이 표시된다.
• 선택한 태그가 없으면 필터 버튼 텍스트는 `태그`이다.
• 태그 1개를 선택하면 필터 버튼 텍스트는 선택한 태그명이다.
• 태그를 2개 이상 선택하면 필터 버튼 텍스트는 `첫 번째 태그 +추가 개수` 형식이다.

**Option 태그 다중 선택**

• 태그 필터를 열면 `태그 검색` 입력과 태그 체크박스 목록이 표시된다.
• 저장된 태그가 없으면 `등록된 태그가 없습니다`를 표시한다.
• 검색 결과가 없으면 `조건과 일치하는 결과가 없습니다`를 표시한다.
• 태그 검색은 입력값을 포함하는 태그를 대소문자 구분 없이 표시한다.
• 여러 태그를 동시에 선택할 수 있다.

**List 필터 결과**

• 선택된 태그가 없으면 태그 조건 없이 회원 목록을 표시한다.
• 태그가 선택되면 보호자 태그 또는 반려견 태그 중 하나라도 선택 태그와 일치하는 회원 행을 표시한다.
• 회원 행에는 반려견 태그가 있을 때 최대 3개까지 칩으로 표시된다.
• 표시 가능한 태그보다 더 많은 태그가 있으면 `+남은 개수` 칩을 표시한다.
• 필터 결과가 없으면 `조건과 일치하는 결과가 없습니다.`를 표시한다.

**Button 초기화**

• `초기화` 버튼을 누르면 선택 태그와 태그 검색어가 모두 비워지고 태그 메뉴가 닫힌다.

**META / Case**

| Case | 모바일 바텀시트에서 태그를 선택해 회원 목록을 필터링한다 |

**Section 태그 바텀시트**

• 모바일 회원 목록 상단에는 태그 필터 버튼과 회원 검색 입력이 함께 표시된다.
• 태그 필터 버튼을 누르면 제목 `태그`가 있는 바텀시트가 열린다.
• 바텀시트 헤더에는 태그 관리 아이콘 버튼과 `닫기` 버튼이 표시된다.
• `닫기` 버튼을 누르면 바텀시트가 닫힌다.

**Input 태그 검색**

• 바텀시트 검색 입력 placeholder는 `태그 검색`이다.
• 선택 태그가 있으면 검색 입력 앞에 선택 태그 칩이 표시된다.
• 선택 태그가 있으면 `×` 전체 해제 버튼이 표시되고, 누르면 선택 태그와 검색어가 모두 비워진다.

**List 필터 결과**

• 태그 선택과 필터 결과 정책은 Web 회원 목록 태그 필터와 동일하다.

## Spec Panel 8

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 태그 관리 |
| 화면 경로 | 회원 > 회원 목록 > 태그 관리 |
| Case | 저장된 태그명을 수정하거나 삭제해 전체 회원 데이터에 반영한다 |

**Section 태그 관리 모달**

• 태그 관리 진입 시 `태그 관리` 모달이 열린다.
• 저장된 태그가 없으면 `등록된 태그가 없습니다`를 표시한다.
• 저장된 태그가 있으면 태그별로 `태그명` 입력 필드와 `삭제` 버튼이 표시된다.

**Button 삭제/복원**

• `삭제` 버튼을 누르면 해당 태그 행은 삭제 상태가 되고 입력 필드는 비활성화된다.
• 삭제 상태의 행에는 `복원` 버튼이 표시된다.
• `복원` 버튼을 누르면 해당 태그 행이 다시 편집 상태가 된다.

**Button 저장/취소**

• `취소` 또는 닫기 버튼을 누르면 편집 중인 태그 변경 내용은 반영되지 않고 모달이 닫힌다.
• `저장` 버튼을 누르면 태그명 변경과 삭제가 공통 태그 목록에 반영된다.
• `저장` 버튼을 누르면 모든 회원의 보호자 태그와 반려견 태그에도 같은 변경이 동기화된다.
• 삭제된 태그는 회원 데이터와 현재 선택된 필터 태그 목록에서 제거된다.
• 저장 성공 시 모달이 닫히고 `태그를 저장했습니다.` 토스트가 표시된다.
• 저장 실패 시 `태그 저장에 실패했습니다. 다시 시도해 주세요.` 토스트가 표시된다.
• 빈 태그명이나 중복 태그명은 저장 시 제외되거나 중복 제거된 태그 목록으로 정리된다.

## Spec Panel 9

**META / Case**

| 항목 | 내용 |
|---|---|
| 플랫폼 | Web |
| 유저 | 비즈 |
| 화면명 | 회원 상세 |
| 화면 경로 | 회원 > 회원 상세 |
| Case | 회원 상세에서 보호자 태그와 반려견 태그를 확인하고 수정한다 |

**Section 태그 표시**

• 회원 상세 카드 헤더와 정보 영역에는 태그가 있을 때만 태그 칩 목록이 표시된다.
• 보호자 태그가 있으면 보호자 정보 관련 영역에 태그 칩을 표시한다.
• 반려견 태그가 있으면 반려견 정보 관련 영역에 태그 칩을 표시한다.
• 태그가 없으면 해당 태그 표시 섹션은 표시되지 않는다.

**Button 보호자 수정**

• `보호자 수정` 모달의 `태그` 항목에서 보호자 태그를 수정할 수 있다.

**Section 예약 태그 필터**

• 유치원 예약 검색 필터에는 태그 필터 버튼이 표시된다.
• 선택한 태그가 없으면 버튼 텍스트는 `태그`이다.
• 태그 1개를 선택하면 버튼 텍스트는 선택한 태그명이다.
• 태그를 2개 이상 선택하면 버튼 텍스트는 `첫 번째 태그 +추가 개수` 형식이다.

**Option 태그 다중 선택**

• 태그 필터 메뉴에는 `태그 검색` 입력과 태그 체크박스 목록이 표시된다.
• 저장된 태그가 없으면 `등록된 태그가 없습니다`를 표시한다.
• 검색 결과가 없으면 `조건과 일치하는 결과가 없습니다`를 표시한다.
• 선택 태그가 있으면 선택 태그 칩과 `×` 전체 해제 버튼을 표시한다.

**List 예약 필터 결과**

• 예약 데이터는 저장된 회원 데이터와 연결되어 보호자 태그와 반려견 태그를 함께 가진다.
• 태그가 선택되면 예약의 보호자 태그와 반려견 태그가 선택 태그를 모두 포함하는 예약만 표시된다.
• 태그 선택이 없으면 태그 조건 없이 예약을 표시한다.

## 정책 확인 필요 항목

1. 태그 입력 제한
   - 확인된 사실: 코드에서는 공백 정리, 빈 값 제외, 대소문자 무시 중복 제외만 확인된다.
   - 결정이 필요한 정책: 태그 글자 수, 허용 문자, 최대 선택 개수 제한을 둘지 확인이 필요하다.
   - 추천 기본값: 현재 프로토타입과 동일하게 별도 제한 없이 저장 전 정리 정책만 적용한다.

2. 회원 목록과 예약 목록의 다중 태그 필터 조건
   - 확인된 사실: 회원 목록은 선택 태그 중 하나라도 일치하면 표시하고, 유치원/호텔링 예약 목록은 선택 태그를 모두 포함해야 표시한다.
   - 결정이 필요한 정책: 회원 목록과 예약 목록의 다중 태그 필터 조건을 서로 다르게 유지할지 확인이 필요하다.
   - 추천 기본값: 현재 프로토타입 동작을 유지한다.
